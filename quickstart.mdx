DefinitelyTyped makes it easy to add TypeScript type safety to JavaScript libraries that don't ship their own types. This guide walks you through installing types and using them in a TypeScript project.

## Install types for a package

To install type definitions for a package, use your package manager with the `@types/` prefix:

<CodeGroup>
```bash npm
npm install --save-dev @types/node
```
```bash yarn
yarn add --dev @types/node
```
```bash pnpm
pnpm add --save-dev @types/node
```
</CodeGroup>

After installation, TypeScript automatically includes the types. You may need to add a reference if you're not using ES modules:

```typescript
/// <reference types="node" />
```

## Check if a package has types

There are three ways a package can provide TypeScript types:

1. **Bundled types** — The package includes a `types` or `typings` field in its `package.json`. No extra install needed.
2. **DefinitelyTyped** — Install `@types/<package-name>` separately.
3. **No types available** — You may need to write your own or use `declare module`.

<Tip>
Visit [npmjs.com](https://www.npmjs.com) and search for your package. If it has a TypeScript icon (TS) badge, types are bundled. If it has a DT badge, types are available via DefinitelyTyped.
</Tip>

## Quick example: using @types/node

<Steps>
  <Step title="Create a TypeScript project">
    ```bash
    mkdir my-project && cd my-project
    npm init -y
    npm install typescript --save-dev
    npx tsc --init
    ```
  </Step>
  <Step title="Install Node.js types">
    ```bash
    npm install --save-dev @types/node
    ```
  </Step>
  <Step title="Write typed Node.js code">
    Create `index.ts` with fully typed Node.js APIs:
    ```typescript index.ts
    import * as fs from "fs";
    import * as path from "path";

    const filePath: string = path.join(__dirname, "hello.txt");
    fs.writeFileSync(filePath, "Hello from TypeScript!");

    const content: string = fs.readFileSync(filePath, "utf-8");
    console.log(content);
    ```
  </Step>
  <Step title="Compile and run">
    ```bash
    npx tsc
    node index.js
    ```
  </Step>
</Steps>

## Find the right package name

For most packages, the `@types` package name matches the npm package name:

| npm package | @types package |
|-------------|----------------|
| `lodash` | `@types/lodash` |
| `react` | `@types/react` |
| `express` | `@types/express` |
| `node` | `@types/node` |

For **scoped packages** (like `@babel/core`), the `@` and `/` are replaced with double underscore:

```bash
npm install --save-dev @types/babel__core
```

See [Scoped packages](/using-types/scoped-packages) for more details.

## Older TypeScript versions

DefinitelyTyped only tests packages on TypeScript versions that are less than 2 years old. For older TypeScript versions, `@types` packages have dist-tags you can target:

```bash
# See available TypeScript version tags for a package
npm dist-tags @types/react
```

This shows tags like `ts2.5`, `ts2.6`, etc., which you can install for compatibility with older TypeScript versions.
