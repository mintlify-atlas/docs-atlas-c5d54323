DefinitelyTyped is one of the most active repositories on GitHub, hosting TypeScript type definitions for thousands of JavaScript libraries. Type definitions contributed here are automatically published to the `@types` scope on npm, making them available to TypeScript developers worldwide.

## How it works

When a JavaScript library doesn't include TypeScript types, DefinitelyTyped fills the gap. Contributors write `.d.ts` declaration files that describe the library's API, and these are published as `@types/<package-name>` on npm. TypeScript automatically picks up these types when you install them.

## The @types ecosystem

The `master` branch is automatically published to the `@types` scope on npm by [DefinitelyTyped-tools](https://github.com/microsoft/DefinitelyTyped-tools/tree/master/packages/publisher). Each package in the `types/` directory of the repository becomes a separate npm package under `@types/`.

For example:
- `types/node/` → `@types/node` on npm
- `types/react/` → `@types/react` on npm
- `types/lodash/` → `@types/lodash` on npm

## Contribution philosophy

DefinitelyTyped's goal is to include type definitions for packages that are actually in use, not every package on npm. Before submitting a new definition, you should intend to use it in your own project.

<Tip>
If you are a library author and your package is written in TypeScript, consider [bundling declaration files](https://www.typescriptlang.org/docs/handbook/declaration-files/publishing.html) in your package directly instead of contributing to DefinitelyTyped.
</Tip>
